# Задача - установить CMake и Boost
## Варианты установки
1. [Установка CMake](https://cmake.org/) с официального источника 
2. [Boost](https://www.boost.org/) с официального источника

С установкой CMake проблем не возникло, тогда как Boost оказался заблокирован на территории РФ
Были предприняты следующие пути решения проблемы: установка boost через пакет vcpkg 
## Ошибки использования vcpkg для установки Boost
1. vcpkg для корретной работы требует PowerShell версии 7.5.1, в свою очередь данная версия PowerShell глючно работает с установленной у меня ОС Windows 10 Pro.
2. При установки более старой версии PowerShell 7.4.7 (подходящей для ОС) происходит сбой вызова, vcpkg требует использования именно новой версии PowerShell 7.5.1

Что происходит когда vcpkg не находит нужной версии PowerShell? Он подтягивает свой внутренний PowerShell 7.5.1
И как выяснилось внутренний PowerShell менеджера пакетов vcpkg имеет поломанные настройки, а именно:
- pws.exe не запускается
- в папке PowerShell лежит поломанный файл ( 1988 или что-то подобное)
  
Одним из вариантов решения проблемы стало добавление PowerShell 7.4.7 в папку vcpkg и создания символической (?) ссылки  для автоматического переключения с поломанной версии PowerShell версии 7.5.1 на работающую PowerShell 7.4.7 (Доработать)
